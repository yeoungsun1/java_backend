0. 복습
1) 입력 클래스
	Scanner sc = new Scanner(System.in);
	import 단축키 => ctrl + shift + o

2) 입력 메소드
	메소드		반환타입	특징
	sc.next()	String	공백 전까지만 입력, 엔터 소모 안 함
	sc.nextLine()	String	공백 포함 한 줄 입력, 엔터 소모(버퍼 비우기)
	sc.nextInt() 	int	정수 입력, 엔터 소모 안 함
	sc.nextDouble()	double 	실수 입력, 엔터 소모 안 함
	* next(), nextInt(), nextDouble() 후 nextLine() 사용시 엔터가 남아있어 입력을 받지 않게됨
	-> nextLine()을 한 번 더 써서 버퍼 비우기 해야함
	* sc.close()	더 이상 입력 받지 않을 때는 리소스 누수를 방지하기 위해 반드시 입력클래스를
			닫아줘야한다!!

3) 클래스와 객체
	클래스(class) 	공통된 속성과 기능을 묶어놓은 설계도

	구성요소	3가지
	필드(Field)	객체의 속성(이름, 나이, 아이디, 비밀번호 등)
	메소드(Method)	객체의 기능(출력하는 기능 등)
	생성자(Constructor) 객체 생성 시 초기값 설정

	객체(Object)	클래스를 기반으로 만들어진 실체
			메모리에 올라간 것

4) 생성자
	기본 생성자	우리가 생성자를 만들지 않으면 컴파일러가 자동 생성
	매개변수가 있는 생성자	매개변수가 있는 생성자를 우리가 직접 만들면 객체 생성시 
				초기값 설정이 가능함
				단, 매개변수 생성자를 만들면 기본 생성자는 자동생성되지 않음

5) 메소드
	메소드 특징	메소드명 뒤에 소괄호()
			키워드 뒤의 소괄호는 메소드가 아니다
			(ex if(), for() 등은 메소드가 아님)
	메소드 정의	메소드를 정의(만들때)할 때는 메소드 영역(중괄호) 밖에서 정의해야한다
	메소드 호출(사용)	메소드를 호출할 때는 메소드 영역 안에서 호출해야한다

6) 메소드 정의
	리턴타입 메소드명(자료형 매개변수){
		실행할 문장;
		return 리턴값;
	}

7) 메소드 호출방법 4가지
	매개변수	리턴값	호출방법
	x	x	객체명.메소드명();
	o	x	객체명.메소드명(인수);
	x	o	리턴타입 변수명 = 객체명.메소드명();
			syso(객체명.메소드명());
	o	o	리턴타입 변수명 = 객체명.메소드명(인수);
			syso(객체명.메소드명(인수));

1. 제어문
1) 조건문
- if문	
(1) if문
	if(조건식){
		조건식이 참일 때 실행할 문장;
	}
(2) if~else문
	if(조건식){
		조건식의 결과가 참일 때 실행될 문장;
	}else{
		조건식의 결과가 거짓일 때 실행될 문장;
	}
(3) if~else if~else문
	if(조건식1){
		조건식1의 결과가 참일 때 실행될 문장;
	}else if(조건식2){
		조건식1의 결과가 거짓이고 조건식2의 결과가 참일 때 실행될 문장;
	}else if(조건식3){
		조건식1과 조건식2의 결과가 거짓이고 조건식3의 결과가 참일 때 실행될 문장;
	}else{
		조건식1과 조건식2, 조건식3의 결과가 모두 거짓일 때 실행될 문장;
	}

- switch문
	switch(변수){
	case 값1:
		실행문;
		break; 	//case마다 break문을 반드시 써야한다!
	case 값2:
		실행문;
		break;
	...
	default:
		실행문;
		break;
	}
